# Unit 2 NFR Design Patterns

- Use a closed named-method facade so IDE completion is the workflow catalog.
- Use one omitted-value sentinel and one resolver for all precedence decisions.
- Use immutable replay records and defensive option copies for deterministic
  period execution.
- Use method-specific normalization functions at the public boundary so backend
  option codes do not leak into tutorials.
- Use explicit state guards for every observational surface.
- Use capability mirroring rather than a method-name string dispatcher for
  all-period execution.
- Use generated valid and invalid argument combinations to verify precedence,
  mutual exclusion, ordering, non-mutation, and JSON stability.
