# Business Logic Model

The live public facade is the coverage denominator. It consists of
`PinchProblem`, `PinchWorkspace`, targeting, all-period targeting, components,
design, selected-network views, and plotting. Every non-private callable or
property on those owners maps to exactly one primary tutorial and one declared
execution profile in the canonical CSV manifest.

Tutorial coverage is counted only from executable notebook cells. A manifest
row is valid when its operation exists on the live owner, its primary tutorial
exists, and the operation is referenced in executable code. Notebook execution
coverage is reported separately by profile so optional solver or HPR
environments are never represented as executed when unavailable.
