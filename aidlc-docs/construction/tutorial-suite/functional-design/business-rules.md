# Tutorial Rules

- Import workflow facades only from `OpenPinch`.
- Show a meaningful result in the first two code cells.
- Use explicit named methods and engineering kwargs, never internal enums or
  closed-string selectors.
- Cache results instead of repeating numerical workflows.
- Use at least two ordered periods in every multiperiod study.
- Preserve native multi-segment input.
- Store no execution count or output in packaged notebooks.
- Declare level, optional extras, expected runtime, and execution profile.
